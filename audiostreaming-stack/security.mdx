## Reporting a vulnerability

**Do not open a public GitHub issue for security vulnerabilities.**

You can report a vulnerability through any of the following channels:

<CardGroup cols={3}>
  <Card title="Web form" icon="browser" href="https://sonicverse.eu/security">
    Submit a report through the dedicated security page on sonicverse.eu.
  </Card>
  <Card title="Email" icon="envelope" href="mailto:security@sonicverse.eu">
    Send details directly to security@sonicverse.eu.
  </Card>
  <Card title="GitHub" icon="github" href="https://github.com/sonicverse-eu/audiostreaming-stack/security/advisories/new">
    Use GitHub's private vulnerability reporting.
  </Card>
</CardGroup>

Include the following in your report:

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Affected version, environment, or commit (if known)
- Any suggested fix or mitigation

You can expect an acknowledgment within **3 business days** and initial triage within **7 days**. Critical issues receive a patch within **14 days**.

## Scope

**In scope:**

- Command injection via the status panel API
- Authentication bypass in the Appwrite integration
- Container escape via Docker socket exposure
- Secrets leakage through API responses
- CORS misconfigurations allowing unauthorized cross-origin access

**Out of scope:**

- Issues requiring physical access to the server
- Denial of service attacks against a specific deployment
- Social engineering
- Vulnerabilities in third-party dependencies (report those upstream)

## Container hardening

All service containers (Status API, Nginx, Analytics, and Icecast) run as non-root users (UID 1000) by default. This limits the impact of a container escape by ensuring processes inside the container do not have root privileges on the host.

## Automated code analysis

Pull requests and pushes to `main` are automatically analyzed by [SonarQube Cloud](https://sonarcloud.io) for code quality, security vulnerabilities, and coverage reporting.

## Secret scanning

This repository uses [TruffleHog](https://trufflesecurity.com/open-source/trufflehog) to detect accidentally committed secrets and sensitive data.

A configuration file is provided at [`trufflehog.toml`](https://github.com/sonicverse-eu/docs/blob/main/trufflehog.toml) in the repository root.

### Running TruffleHog locally

Install TruffleHog:

```bash
# macOS
brew install trufflehog

# Linux / CI (latest release binary)
curl -sSfL https://raw.githubusercontent.com/trufflesecurity/trufflehog/main/scripts/install.sh | sh -s -- -b /usr/local/bin
```

Scan the repository against the full git history:

```bash
trufflehog git file://. --config trufflehog.toml
```

Scan only the current working tree (faster, good for pre-commit checks):

```bash
trufflehog filesystem . --config trufflehog.toml
```

### Running in CI

Add TruffleHog to your GitHub Actions workflow:

```yaml
- name: Scan for secrets
  uses: trufflesecurity/trufflehog@main
  with:
    extra_args: --config trufflehog.toml
```
