<Update label="2026-04-06" description="v1.4.0" tags={["Release"]}>
  ## Security hardening and dependency updates

  This release focuses on supply-chain security, container hardening, and automated code review tooling.

  ### New features

  - **CodeRabbit automated code review** â€” Pull requests targeting `main` now receive automated reviews from [CodeRabbit](https://coderabbit.ai) with per-path review instructions tuned to each service's tooling (Ruff for Python, ESLint for TypeScript, hadolint for Dockerfiles).
  - **TruffleHog secret scanning** â€” A CI workflow scans every push and pull request for accidentally committed secrets. See [security](/audiostreaming-stack/security) for local usage instructions.
  - **Non-root containers** â€” The Status API, Nginx, Analytics, and Icecast containers now run as non-root users (UID 1000) by default, reducing the blast radius of container escapes.

  ### Security updates

  - Upgraded Nginx base image from `alpine` to `stable-alpine-slim` (pinned)
  - Upgraded Liquidsoap base image to latest `v2.4.x` release
  - Patched `libsndfile`, `libfreetype6`, `libpng`, `openssl-provider-legacy`, and `dpkg` in the Liquidsoap image
  - Upgraded Werkzeug from 2.3.8 to 3.1.6, fixing debugger RCE and multipart DoS vulnerabilities
  - Upgraded requests from 2.32.2 to 2.33.0, fixing credential leakage via `.netrc` parsing
  - Upgraded gunicorn from 22.0.0 to 23.0.0, fixing HTTP request splitting
  - Removed `docker-cli` from the Status API image in favor of the Python Docker SDK, reducing the attack surface

  **Full changelog**: [v1.4.0](https://github.com/sonicverse-eu/audiostreaming-stack/releases/tag/v1.4.0)
</Update>

<Update label="2026-04-06" description="v1.1.1" tags={["Release"]}>
  ## Docker Hub as primary registry and Code of Conduct

  ### New features

  - **Docker Hub as primary registry** â€” Pre-built container images are now published to Docker Hub as the primary registry (`docker.io/sonicverse/...`). GHCR remains available as a mirror. The installer and `docker-compose.yml` pull from Docker Hub by default, which provides faster downloads for most users. See [container registries](#container-registries) below for image names.
  - **Code of Conduct** â€” A [Code of Conduct](/audiostreaming-stack/code-of-conduct) now governs community interactions. All contributors and participants are expected to follow these guidelines.
  - **Blacksmith CI runners** â€” GitHub Actions workflows now run on Blacksmith runners for faster builds, automatic Docker layer caching, and improved observability.

  ### Updates

  - The `docker-compose.yml` now references Docker Hub image names (`docker.io/sonicverse/audiostreaming-stack-*`) instead of GHCR.
  - The CI pipeline validates that images published to Docker Hub and GHCR have matching digests across `linux/amd64`, `linux/arm64`, and `linux/386` platforms, using per-platform digest comparison for more robust verification.
  - Contributing guidelines now reference the Code of Conduct.

  ### Container registries

  | Registry | Image prefix | Role |
  |---|---|---|
  | Docker Hub | `docker.io/sonicverse/audiostreaming-stack-*` | Primary (default) |
  | GHCR | `ghcr.io/sonicverse-eu/audiostreaming-stack/*` | Mirror |

  If you previously pinned images to GHCR, they continue to work. No action is required for existing deployments using the default configuration â€” `docker compose pull` fetches the latest images from Docker Hub automatically.

  **Full changelog**: [v1.1.1](https://github.com/sonicverse-eu/audiostreaming-stack/releases/tag/v1.1.1)
</Update>

<Update label="2026-04-06" description="v1.1.0" tags={["Release"]}>
  ## Easier installation and flexible deployment

  This release focuses on making the stack faster to deploy and easier to manage.

  ### New features

  - **One-line remote install** â€” You can now install the stack with a single `curl | bash` command, no need to manually clone the repository first. The installer detects remote execution and handles everything automatically.
  - **Pre-built container images** â€” Deploy using pre-built images from Docker Hub instead of building locally. This significantly reduces setup time on low-powered servers. See the [quickstart](/audiostreaming-stack/quickstart) for details.
  - **Minimal deployment mode** â€” The installer now supports a minimal mode that starts only the core streaming services. It skips optional components like analytics and the status API, making it ideal for quick testing or resource-constrained environments.
  - **Development mode** â€” A new local build mode for contributors who want to build and test container images on their own machine.
  - **Root-level dependency installer** â€” A single script installs all development dependencies from the project root. Contributors no longer need to enter each service directory individually.

  ### Updates

  - The installer now displays clearer step-by-step output with dynamic step numbering. This makes it easier to follow progress during setup.
  - The `.env.example` file now includes better descriptions and clearer defaults, so you know exactly which variables a minimal deployment requires. See [configuration](/audiostreaming-stack/configuration) for details.
  - The Status Panel service is now called **Status API** across the project for consistency. No action is needed if you use the default configuration.
  - A community Slack channel is now available for support and collaboration. Look for the invite link in the project README.

  **Full changelog**: [v1.1.0](https://github.com/sonicverse-eu/audiostreaming-stack/releases/tag/v1.1.0)
</Update>
